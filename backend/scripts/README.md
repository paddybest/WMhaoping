# 数据迁移脚本

本目录包含从微信云开发迁移到独立后端的数据迁移脚本。

## 文件说明

- `migrate-data.ts` - TypeScript 版本的迁移脚本
- `migrate-data.js` - JavaScript 版本的迁移脚本（编译后）
- `init.sql` - 数据库初始化脚本

## 迁移内容

1. **商品数据** (products)
   - 商品名称
   - 标签数组
   - 创建和更新时间

2. **用户数据** (users)
   - 微信 openid
   - 昵称
   - 头像链接

3. **奖品数据** (prizes)
   - 奖品名称
   - 描述
   - 中奖概率
   - 库存数量
   - 图片链接

4. **评价数据** (reviews)
   - 用户ID
   - 商品ID
   - 评价内容
   - 评分
   - 标签
   - 图片链接

5. **抽奖数据** (lottery_records, lottery_codes)
   - 用户ID
   - 奖品ID
   - 奖品名称
   - 兑换码
   - 核销状态
   - 创建和核销时间

## 使用方法

### 1. 准备工作

确保以下条件满足：
- MySQL 数据库已启动并配置正确
- Redis 服务已启动
- 后端服务已安装依赖
- 环境变量已配置（查看 .env 文件）

### 2. 运行迁移脚本

```bash
# 进入后端目录
cd backend

# 编译 TypeScript（如果使用 TypeScript 版本）
npm run build

# 运行迁移脚本
node scripts/migrate-data.js
```

### 3. 验证迁移

脚本运行后会自动验证迁移结果，输出各类数据的数量统计。

## 注意事项

1. **数据备份**
   - 运行迁移前请务必备份数据库
   - 建议在测试环境先进行迁移测试

2. **图片迁移**
   - 当前脚本只记录图片迁移日志
   - 需要手动完成从微信云存储到阿里云OSS的图片上传
   - 图片映射关系需要根据实际情况调整

3. **重复数据**
   - 脚本使用 `ON DUPLICATE KEY UPDATE` 避免重复插入
   - 已存在的数据会被更新，不会导致错误

4. **错误处理**
   - 单个数据迁移失败不会影响其他数据
   - 会在控制台输出详细的错误信息

## 自定义迁移

如需迁移实际的生产数据，请：

1. 修改 `migrate-products`、`migrate-users` 等函数中的数据源
2. 从微信云导出实际数据
3. 根据实际数据结构调整迁移逻辑
4. 更新图片映射关系

## 故障排除

### 常见错误

1. **连接数据库失败**
   - 检查 MySQL 服务是否启动
   - 验证数据库连接配置（.env 文件）

2. **数据插入失败**
   - 检查数据格式是否正确
   - 确认字段类型匹配

3. **依赖缺失**
   - 运行 `npm install` 安装依赖

### 日志查看

脚本会输出详细的迁移日志，包括：
- 成功迁移的记录数
- 失败的记录及原因
- 验证结果统计

## 回滚方案

如果迁移出现问题，可以：

1. 使用数据库备份恢复
2. 清理相关表后重新运行迁移
3. 检查具体错误并修复后重新执行
