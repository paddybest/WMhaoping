# 部署检查清单

## 部署前检查

### [ ] 代码质量检查
- [ ] 所有测试通过
- [ ] 代码覆盖率 >= 80%
- [ ] ESLint 检查通过
- [ ] TypeScript 编译通过

### [ ] 安全检查
- [ ] 依赖漏洞扫描（npm audit）
- [ ] 环境变量密钥已更新（生产环境）
- [ ] JWT 密钥长度 >= 32 字符
- [ ] CORS 域名配置正确

### [ ] 基础设施检查
- [ ] 数据库连接正常
- [ ] Redis 连接正常
- [ ] 文件存储（OSS）配置正确
- [ ] 微信小程序配置正确

### [ ] 监控配置
- [ ] 日志配置正确
- [ ] 告警通知配置
- [ ] 健康检查配置
- [ ] 监控指标收集配置

## 灰度发布步骤

### 第1步：准备阶段
- [ ] 创建备份（代码、数据库、配置文件）
- [ ] 更新版本号
- [ ] 更新 CHANGELOG.md
- [ ] 通知团队成员

### 第2步：内部测试环境部署
- [ ] 部署到内部测试环境
- [ ] 运行回归测试
- [ ] 验证所有核心功能
- [ ] 监控系统性能指标

### 第3步：小流量发布
- [ ] 配置流量分配（10% → 30% → 50%）
- [ ] 发布到生产环境的 10% 流量
- [ ] 监控关键指标（错误率、响应时间、CPU/内存）
- [ ] 准备回滚方案

### 第4步：逐步放量
- [ ] 错误率 < 0.1% 时，逐步增加到 30%
- [ ] 错误率 < 0.05% 时，逐步增加到 50%
- [ ] 错误率 < 0.01% 时，逐步增加到 100%

### 第5步：全量发布
- [ ] 监控 24 小时系统稳定性
- [ ] 收集用户反馈
- [ ] 观察数据库性能
- [ ] 监控 API 响应时间

## 监控指标

### 关键指标
- [ ] HTTP 错误率 < 0.1%
- [ ] 平均响应时间 < 200ms
- [ ] 99% 响应时间 < 500ms
- [ ] CPU 使用率 < 70%
- [ ] 内存使用率 < 80%
- [ ] 数据库连接数正常

### 告警阈值
- [ ] 错误率 > 1%（严重告警）
- [ ] 响应时间 > 1s（警告）
- [ ] CPU 使用率 > 90%（严重告警）
- [ ] 内存使用率 > 95%（严重告警）
- [ ] 数据库连接 > 90%（警告）

## 回滚条件

以下情况必须立即回滚：
- [ ] 错误率 > 5%
- [ ] 数据库连接失败
- [ ] 内存泄漏（持续增长）
- [ ] 关键 API 不可用
- [ ] 用户体验严重下降

## 回滚步骤
1. 立即停止新版本部署
2. 恢复备份的服务器
3. 切换流量到旧版本
4. 通知用户系统维护
5. 分析问题并修复

## 上线后任务
- [ ] 收集用户反馈
- [ ] 性能基准测试
- [ ] 更新文档
- [ ] 归档部署记录
- [ ] 团队复盘会议

## 注意事项
1. **灰度发布必须在业务低峰期进行**
2. **确保监控告警系统正常工作**
3. **准备好 24 小时值班支持**
4. **提前通知所有相关人员**
5. **保持用户沟通渠道畅通**
