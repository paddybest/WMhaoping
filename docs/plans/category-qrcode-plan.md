# 二级分类二维码功能实施计划

## 需求概述
商家商品支持二级目录，然后给第二级目录生成二维码，用户扫描二维码后可以直接选择具体商品进行评价。

## 数据结构
```
一级分类 (level=0)
  └── 二级分类 (level=1)
        └── 商品 (product_items.category_id = 二级分类ID)
```

## 任务列表

### 任务1: 修改商家端分类管理，限制为二级结构
- 修改商家端分类管理页面 (Categories.tsx)
- 添加父分类选择器，只允许选择一级分类作为父分类
- 创建/编辑时限制只能创建二级分类

### 任务2: 确保商品关联到二级分类
- 检查商品创建逻辑，确保商品只能关联到二级分类（叶子节点）
- 可能需要修改 ProductEditModal.tsx 中的分类选择器

### 任务3: 后端新增分类二维码API
- 新增 `GET /api/merchant/categories/level2` - 获取所有二级分类
- 新增 `POST /api/merchant/categories/:id/qrcode` - 生成分类二维码
- 复用现有的二维码生成服务

### 任务4: 商家端新增分类二维码管理页面
- 创建 CategoryQRCodeManager.tsx 页面
- 显示所有二级分类列表
- 支持单个/批量生成二维码
- 支持下载二维码

### 任务5: 小程序端解析URL参数并显示商品列表
- 修改 index.js 的 onLoad 方法
- 当有 categoryId 参数时，加载该分类下的商品列表
- 让用户选择具体商品后再选择标签和评分

## 预期效果
1. 商家创建分类时只能创建一级或二级分类
2. 二级分类可以生成专属二维码
3. 用户扫码后直接显示该分类下的商品列表
4. 用户选择商品后进行评价生成
