# 外卖好评宝 - 阿里云上线部署指南

## 概述

本文档详细介绍如何将外卖好评宝项目部署到阿里云服务器，包括后端 API 服务、前端商家端和微信小程序的部署配置。

## 准备工作

### 1. 阿里云资源清单

| 资源类型 | 规格要求 | 说明 |
|---------|---------|------|
| ECS 服务器 | 2核4G起步 | 推荐使用 CentOS 8 / Ubuntu 22.04 |
| RDS MySQL | 2核4G 50GB | 数据库实例 |
| OSS 对象存储 | 按量付费 | 存储图片和文件 |
| 安全组 | 开放端口 | 80, 443, 8080, 22 |

### 2. 域名准备

- 购买域名并完成备案（如使用中国大陆服务器）
- 配置域名解析到服务器公网 IP

## 第一部分：服务器环境配置

### 1.1 系统初始化

```bash
# 更新系统
yum update -y  # CentOS
# 或
apt update && apt upgrade -y  # Ubuntu

# 安装基础工具
yum install -y git curl wget nginx vim  # CentOS
# 或
apt install -y git curl wget nginx vim   # Ubuntu

# 安装 Node.js 18.x
curl -fsSL https://rpm.nodesource.com/setup_18.x | bash -
yum install -y nodejs
# 验证
node -v  # 应显示 v18.x.x
```

### 1.2 安装 Docker

```bash
# 安装 Docker
curl -fsSL https://get.docker.com | sh

# 启动并设置开机自启
systemctl start docker
systemctl enable docker

# 验证
docker --version
```

### 1.3 安装 Docker Compose

```bash
# 下载 Docker Compose
curl -L "https://github.com/docker/compose/releases/download/v2.24.0/docker-compose-$(uname -m)" -o /usr/local/bin/docker-compose

# 添加执行权限
chmod +x /usr/local/bin/docker-compose

# 验证
docker-compose --version
```

## 第二部分：数据库配置

### 2.1 创建 RDS 数据库

1. 登录阿里云控制台
2. 进入 RDS MySQL 产品页面
3. 创建实例（选择 2核4G 规格）
4. 设置账号密码
5. 创建数据库 `haopingbao`

```sql
-- 连接 RDS 后执行
CREATE DATABASE haopingbao CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

### 2.2 配置数据库连接

在项目后端 `.env` 文件中配置：

```env
# 数据库配置
DB_HOST=your-rds-endpoint.mysql.rds.aliyuncs.com
DB_PORT=3306
DB_USER=your_db_user
DB_PASSWORD=your_db_password
DB_NAME=haopingbao
```

### 3.1 创建 OSS Bucket

1. 登录阿里云控制台


## 第四部分：后端服务部署

### 4.1 项目配置

在 `backend/.env` 中配置生产环境变量：

```env
# 服务端口
PORT=8080
NODE_ENV=production

# 数据库
DB_HOST=your-rds-endpoint.mysql.rds.aliyuncs.com
DB_PORT=3306
DB_USER=your_db_user
DB_PASSWORD=your_db_password
DB_NAME=haopingbao

# JWT 密钥（生产环境使用强密钥）
JWT_SECRET=your-very-long-random-secret-key-at-least-32-chars

# OSS 存储
OSS_ACCESS_KEY_ID=your_access_key_id
OSS_ACCESS_KEY_SECRET=your_access_key_secret
OSS_BUCKET=your-bucket-name
OSS_REGION=oss-cn-hangzhou
OSS_ENDPOINT=oss-cn-hangzhou.aliyuncs.com

# 阿里云通义千问（AI 生成评价）
WANXIANG_API_KEY=your-wanxiang-api-key
```

### 4.2 部署后端

```bash
# 进入后端目录
cd /opt/haopingbao/backend

# 安装依赖
npm install --production

# 编译 TypeScript
npm run build

# 启动服务（使用 PM2）
npm install -g pm2
pm2 start dist/server.js --name haopingbao-backend

# 设置开机自启
pm2 startup
pm2 save
```

### 4.3 使用 Docker 部署（推荐）

创建 `backend/Dockerfile`:

```dockerfile
FROM node:18-alpine

WORKDIR /app

# 安装依赖
COPY package*.json ./
RUN npm ci --only=production

# 编译 TypeScript
COPY tsconfig.json ./
COPY src ./src
RUN npm run build

# 复制配置文件
COPY .env ./

# 暴露端口
EXPOSE 8080

# 启动服务
CMD ["node", "dist/server.js"]
```

创建 `docker-compose.yml`:

```yaml
version: '3.8'

services:
  backend:
    build: .
    ports:
      - "8080:8080"
    environment:
      - NODE_ENV=production
    env_file:
      - .env
    restart: unless-stopped
    networks:
      - haopingbao-network

networks:
  haopingbao-network:
    driver: bridge
```

启动：
```bash
docker-compose up -d
```

## 第五部分：前端商家端部署

### 5.1 构建生产版本

在本地构建（由于需要处理环境变量）：

```bash
cd shangjiaduan

# 创建生产环境配置
echo "VITE_API_BASE_URL=https://your-domain.com/api" > .env.production

# 构建
npm run build
```

### 5.2 配置 Nginx

将构建产物上传到服务器：

```bash
# 上传构建文件
scp -r dist/* user@your-server:/usr/share/nginx/html/
```

配置 Nginx `/etc/nginx/conf.d/haopingbao.conf`:

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 前端静态文件
    location / {
        root /usr/share/nginx/html;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # API 代理
    location /api {
        proxy_pass http://127.0.0.1:8080;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_cache_bypass $http_upgrade;
    }

    # WebSocket 支持
    location /ws {
        proxy_pass http://127.0.0.1:8080;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
    }
}
```

### 5.3 配置 HTTPS（推荐）

使用 Let's Encrypt 免费证书：

```bash
# 安装 Certbot
yum install -y certbot python3-certbot-nginx

# 获取证书
certbot --nginx -d your-domain.com

# 自动续期
certbot renew --dry-run
```

## 第六部分：Git 自动化部署（可选）

### 6.1 配置 Git 仓库

```bash
# 在服务器创建 Git 仓库
cd /opt/haopingbao
git init --bare

# 创建部署钩子
cat > hooks/post-receive << 'EOF'
#!/bin/bash

GIT_DIR=/opt/haopingbao
BRANCH=master

while read oldrev newrev refname; do
    if [ "$refname" = "refs/heads/$BRANCH" ]; then
        echo "Deploying to production..."
        cd /opt/haopingbao
        git --work-tree=/opt/haopingbao-backend checkout -f $BRANCH

        # 重启后端服务
        cd /opt/haopingbao-backend
        docker-compose down
        docker-compose up -d --build
    fi
done
EOF

chmod +x hooks/post-receive
```

### 6.2 本地配置远程仓库

```bash
# 添加远程仓库
git remote add production user@your-server:/opt/haopingbao

# 部署
git push production master
```

## 第七部分：安全配置

### 7.1 防火墙配置

```bash
# 配置防火墙
firewall-cmd --permanent --add-port=22/tcp   # SSH
firewall-cmd --permanent --add-port=80/tcp   # HTTP
firewall-cmd --permanent --add-port=443/tcp  # HTTPS
firewall-cmd --reload
```

### 7.2 安全组配置

在阿里云控制台配置安全组：

| 协议 | 端口 | 源 | 说明 |
|-----|------|-----|------|
| TCP | 22 | 0.0.0.0/0 | SSH |
| TCP | 80 | 0.0.0.0/0 | HTTP |
| TCP | 443 | 0.0.0.0/0 | HTTPS |

### 7.3 定期备份

创建数据库备份脚本 `/opt/backup.sh`:

```bash
#!/bin/bash

DATE=$(date +%Y%m%d_%H%M%S)
BACKUP_DIR=/opt/backups

# 备份数据库
mysqldump -h your-rds-endpoint.mysql.rds.aliyuncs.com \
    -u your_db_user -p'your_db_password' \
    haopingbao > $BACKUP_DIR/db_$DATE.sql

# 删除 7 天前的备份
find $BACKUP_DIR -name "*.sql" -mtime +7 -delete

# 备份上传到 OSS（可选）
ossutil cp $BACKUP_DIR/ oss://your-bucket-name/backups/
```

添加定时任务：

```bash
crontab -e
# 每天凌晨 3 点执行备份
0 3 * * * /opt/backup.sh
```
