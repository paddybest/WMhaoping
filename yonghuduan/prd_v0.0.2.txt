# AI 评价助手小程序客户端产品需求文档 (PRD)

## 1. 项目概述

### 1.1 产品定位
AI 评价助手是一款面向电商买家的辅助工具，旨在通过 AI 智能生成评价文案与配图，降低用户撰写高质量评价的门槛，并提供奖励激励用户参与评价沉淀，最终引导用户进入商家私域客服体系。

### 1.2 技术栈
- **开发框架**: 微信原生框架 (WXML/WXSS/JavaScript)
- **后端服务**: 微信云开发 (CloudBase)
- **数据库**: 云开发阿里云数据库
- **存储**: 云存储 (CDN)
- **云函数**: Node.js 18+
- **AI 接口**: DeepSeek API

### 1.3 目标用户
- 电商平台购物后的买家
- 愿意参与"分享有奖"活动的用户
- 希望快速生成评价文案的用户

## 2. 核心业务流程

### 2.1 用户端完整流程
1. **进入首页**: 选择商品类目
2. **选择标签**: 从预设标签库中选择多个标签
3. **AI生成**: 系统调用AI生成文案并匹配相关图片
4. **查看结果**: 展示生成的文案和配图
5. **保存图片**: 用户保存图片到相册
6. **领取奖励**: 系统生成验证码并引导联系客服

## 3. 数据库设计

### 3.1 集合结构

#### `products` (类目配置表)
| 字段名 | 类型 | 描述 | 访问权限 |
|--------|------|------|----------|
| _id | String | 自动生成ID | 所有用户只读 |
| name | String | 类目名称 | 所有用户只读 |
| tags | Array | 该类目可选标签 | 所有用户只读 |
| updateTime | Date | 最后修改时间 | 所有用户只读 |

#### `images` (素材库)
| 字段名 | 类型 | 描述 |
|--------|------|------|
| _id | String | 自动生成ID |
| fileID | String | 云存储图片ID (格式: cloud://...) |
| categoryId | String | 关联products表的_id |

#### `lottery_codes` (奖励记录)
| 字段名 | 类型 | 描述 | 访问权限 |
|--------|------|------|----------|
| _id | String | 自动生成ID | 仅管理端可读写 |
| code | String | 唯一6位随机码(大写字母+数字) | 仅管理端可读写 |
| status | Number | 状态: 0-未使用, 1-已核销 | 仅管理端可读写 |
| openid | String | 生成该码的用户微信标识 | 仅管理端可读写 |
| createTime | Date | 生成时间 | 仅管理端可读写 |

## 4. 云函数接口规范

### 4.1 `generateReview` 云函数

#### 接口功能
接收用户选择的标签，请求AI生成文案并从图库中匹配相关图片。

#### 请求参数
```json
{
  "categoryId": "string, 必填, 商品类目ID",
  "tags": ["array, 必填, 用户选择的标签数组"]
}
```

#### 处理逻辑
1. **AI文案生成**:
   - 构造Prompt: "你是一个电商买家，根据以下标签：[标签列表]，写一段60字左右的中文评价，语气自然，多使用Emoji，体现真实购物体验。"
   - 调用DeepSeek API生成文案

2. **图片匹配**:
   - 根据categoryId从images集合中随机抽取3张图片
   - 使用云存储fileID构建完整的图片URL

3. **内容安全审核**:
   - 调用微信官方接口 `openapi.security.msgSecCheck`
   - 如审核不通过，使用预设的默认文案替换

#### 响应格式
```json
{
  "success": true,
  "data": {
    "text": "AI生成的评价文案",
    "imgList": [
      "https://xxx.cloudbase.net/fileID1",
      "https://xxx.cloudbase.net/fileID2",
      "https://xxx.cloudbase.net/fileID3"
    ]
  }
}
```

#### 错误响应
```json
{
  "success": false,
  "error": {
    "code": "ERROR_CODE",
    "message": "错误描述"
  }
}
```

### 4.2 `createLotteryCode` 云函数

#### 接口功能
用户保存图片成功后生成唯一的兑换验证码。

#### 请求参数
```json
{
  "openid": "string, 必填, 当前用户的openid"
}
```

#### 处理逻辑
1. 生成6位不重复的随机码（大写字母+数字）
2. 将记录插入lottery_codes集合
3. 设置status为0（未使用）
4. 记录createTime为当前时间

#### 响应格式
```json
{
  "success": true,
  "data": {
    "code": "ABC123"
  }
}
```

## 6. 核心业务逻辑实现

### 6.1 图片保存与奖励闭环逻辑

#### 完整流程
```javascript
async saveAllImagesAndGetReward() {
  try {
    // 1. 检查相册权限
    const authStatus = await this.checkAlbumPermission()
    if (!authStatus) {
      await this.showPermissionGuide()
      return
    }
    
    // 2. 保存所有图片
    let successCount = 0
    for (const imgUrl of this.data.imageUrls) {
      try {
        await this.saveImageToAlbum(imgUrl)
        successCount++
      } catch (error) {
        console.error('保存图片失败:', error)
      }
    }
    
    // 3. 判断保存结果
    if (successCount === this.data.imageUrls.length) {
      // 全部保存成功
      await this.generateAndShowRewardCode()
    } else {
      wx.showToast({
        title: `保存成功${successCount}张`,
        icon: 'none'
      })
    }
  } catch (error) {
    console.error('保存过程出错:', error)
  }
}
```

#### 权限检查
```javascript
checkAlbumPermission() {
  return new Promise((resolve) => {
    wx.getSetting({
      success: (res) => {
        if (res.authSetting['scope.writePhotosAlbum'] === false) {
          // 已拒绝授权
          resolve(false)
        } else {
          resolve(true)
        }
      },
      fail: () => resolve(false)
    })
  })
}
```

#### 权限引导
```javascript
showPermissionGuide() {
  wx.showModal({
    title: '需要相册权限',
    content: '保存图片需要您授权访问相册',
    confirmText: '去设置',
    success: (res) => {
      if (res.confirm) {
        wx.openSetting({
          success: (settingRes) => {
            if (settingRes.authSetting['scope.writePhotosAlbum']) {
              // 用户同意了授权
              this.saveAllImagesAndGetReward()
            }
          }
        })
      }
    }
  })
}
```

#### 生成奖励码
```javascript
async generateAndShowRewardCode() {
  wx.showLoading({ title: '生成中...' })
  
  try {
    // 调用云函数
    const result = await wx.cloud.callFunction({
      name: 'createLotteryCode',
      data: {
        openid: app.globalData.openid
      }
    })
    
    wx.hideLoading()
    
    // 显示奖励弹窗
    if (result.success) {
      this.showRewardModal(result.data.code)
    } else {
      wx.showToast({
        title: '生成验证码失败',
        icon: 'error'
      })
    }
  } catch (error) {
    wx.hideLoading()
    wx.showToast({
      title: '网络错误，请重试',
      icon: 'error'
    })
  }
}
```

### 6.2 异常处理机制

#### 网络异常处理
```javascript
// 统一封装云函数调用
async callCloudFunction(name, data) {
  try {
    const result = await wx.cloud.callFunction({
      name,
      data,
      config: {
        env: this.data.envId,
        timeout: 10000 // 10秒超时
      }
    })
    
    if (result.errMsg === 'cloud.callFunction:ok') {
      return result.result
    } else {
      throw new Error(result.errMsg)
    }
  } catch (error) {
    // 网络错误处理
    if (error.errCode === -404011) {
      // 云函数不存在
      wx.showToast({ title: '服务异常，请稍后重试', icon: 'error' })
    } else if (error.errCode === -404009) {
      // 网络错误
      wx.showToast({ title: '网络异常，请检查网络', icon: 'error' })
    } else {
      wx.showToast({ title: '操作失败，请重试', icon: 'error' })
    }
    throw error
  }
}
```

#### 图片加载失败处理

## 7. 安全与合规规范

### 7.1 内容安全策略
1. **双重审核机制**
   - 云函数中调用微信安全接口
   - 前端可选的二次校验（针对敏感词）

2. **敏感词规避**
   - 避免出现"返现"、"现金"、"刷单"等词汇
   - 使用"神秘礼包"、"体验奖励"等替代词
   - 文案模板中不包含任何支付、交易相关内容

### 7.2 权限管理
1. **数据库权限**
   - products表: 所有用户只读
   - lottery_codes表: 仅管理端可读写

2. **用户隐私**
   - 仅收集必要的openid用于奖励记录
   - 不存储用户个人信息
   - 图片保存仅使用相册写入权限

### 7.3 数据安全
1. **传输安全**
   - 所有API请求使用HTTPS
   - 敏感数据加密传输

2. **存储安全**
   - 用户数据存储在微信云开发
   - 图片资源使用CDN加速和安全防护
