# AI 评价助手小程序项目需求文档 (PRD)

## 1. 项目概览
- **项目名称**：AI 评价助手 (AI Review & Reward System)
- **核心目标**：通过 AI 生成高质量评价文案与配图，降低用户评价门槛，并引导用户沉淀至私域客服。
- **技术路线**：
  - **用户端**：微信原生小程序（Antigravity 生成）
  - **商家端**：Web 后台 (Vue3 + Element Plus，Antigravity 生成)
  - **后端/数据库**：微信云开发 (CloudBase)
  - **逻辑/API**：Claude 3.5 Sonnet / Gemini (通过云函数调用)

---

## 2. 核心业务流程

### 2.1 买家路径 (C端)
1. **进入页面**：选择商品类目（如：女装、数码、零食）。
2. **多选标签**：勾选系统预设的评价关键词（如：物流快、包装好、无异味）。
3. **AI 生成**：点击生成，系统展示 AI 写的文案及数据库匹配的实拍图。
4. **保存奖励**：用户保存图片到相册后，系统触发回调，发放一个 6 位验证码。
5. **兑奖引导**：引导用户复制微信号，联系客服发送验证码领奖。

### 2.2 商家路径 (B端)
1. **基础配置**：在 Web 后台添加商品类目及其对应标签。
2. **素材管理**：批量上传该类目的实拍图片至云存储。
3. **核销逻辑**：用户发来码后，在后台输入验证码进行核销（标记为“已使用”）。

---

## 3. 数据库设计 (Database Schema)

### 3.1 集合：`products` (类目表)
| 字段名 | 类型 | 描述 |
| :--- | :--- | :--- |
| `_id` | String | 自动生成 |
| `name` | String | 类目名称 (如：数码产品) |
| `tags` | Array | 预设标签库 (如：["续航久", "屏幕好"]) |

### 3.2 集合：`images` (素材表)
| 字段名 | 类型 | 描述 |
| :--- | :--- | :--- |
| `_id` | String | 自动生成 |
| `fileID` | String | 云存储图片地址 |
| `categoryId` | String | 关联 products 表的 _id |

### 3.3 集合：`lottery_codes` (核销表)
| 字段名 | 类型 | 描述 |
| :--- | :--- | :--- |
| `code` | String | 唯一 6 位随机验证码 |
| `status` | Number | 0: 未核销, 1: 已核销 |
| `openid` | String | 用户唯一标识 |

---

## 4. 后端接口设计 (云函数)

### 4.1 `generateReview`
- **功能**：调用 AI 并匹配图片。
- **输入**：`categoryId`, `tags[]`
- **逻辑**：
  1. 向 Claude API 发送请求，Prompt 为：“你是一个电商买家，根据[tags]写一段 60 字左右的中文评价，语气自然，多用 Emoji。”
  2. 根据 `categoryId` 从 `images` 集合随机选取 3 张图。
- **返回**：`{ text, imageUrls }`

### 4.2 `createLotteryCode`
- **功能**：生成中奖凭证。
- **逻辑**：生成 6 位随机字符串 -> 存入 `lottery_codes` -> 返回前端。

---

## 5. 前端页面需求 (Antigravity 指令参考)

### 5.1 用户端小程序
- **首页**：类目下拉框 + 多选 Tag 列表 + “立即生成”按钮。
- **结果页**：
  - 文字展示区（带一键复制功能）。
  - 图片展示区（3 张图片网格展示）。
  - **核心动作**：点击“保存图片” -> 调用 `wx.saveImageToPhotosAlbum` -> 成功后弹出“领奖码”及“客服二维码”。

### 5.2 商家 Web 后台
- **上传页**：支持图片上传至 CloudBase 存储，并绑定对应类目。
- **核销页**：一个输入框 + 一个查询按钮，点击后将对应的 `status` 改为 1。

---

## 6. 合规与安全
1. **安全检测**：必须调用微信 `security.msgSecCheck` 接口，过滤 AI 生成的潜在违规文字。
2. **文案规避**：前端 UI 避免出现“好评返现”，建议改为“分享有奖”或“体验官计划”。
3. **相册权限**：妥善处理用户拒绝授权相册的情况，提供重新引导设置的按钮。
